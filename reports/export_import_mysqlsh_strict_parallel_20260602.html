Export/Import Strict Parallel 개선 보고서

작성일: 2026-06-02

핵심 결론

기존 문제는 Import 버튼만의 문제가 아니었다. threads > 1 Export에서 각 worker가 독립적으로 connection을 열어 SELECT를 시작했기 때문에, 여러 worker가 같은 기준 시점의 snapshot을 바라본다고 증명할 수 없었다. 그래서 manifest가 limited_restorable로 내려가는 판단은 맞았고, 빠진 것은 MySQL Shell 방식의 snapshot 동기화 경로였다.

적용한 변경

영역변경 내용
Rust Core Export MySQL 병렬 Export에서 InnoDB 테이블만 대상이고 threads > 1인 경우, FLUSH TABLES WITH READ LOCK 중 worker connection을 미리 열고 START TRANSACTION WITH CONSISTENT SNAPSHOT을 시작한 뒤 LOCK INSTANCE FOR BACKUP을 잡고 병렬 dump를 수행하는 strict 경로를 추가했다. 2026-06-02 추가 수정으로 FLUSH TABLES WITH READ LOCK 권한이 없으면 LOCK TABLES ... READ 대체 경로를 시도하고, 자동 모드에서는 strict 권한 경로가 불가능할 때 Export를 실패시키지 않고 limited_restorable로 명확히 강등한다.
Rust Core Manifest 새 정책 lock_synchronized_transaction_snapshot을 추가했다. 이 정책은 snapshot과 checksum 증거가 완성되고 경고/blocker가 없을 때 strict_restorable로 판정된다.
Import UI strict_restorable은 기존처럼 권장 Import로 진행한다. limited_restorable은 쉬운 한국어 설명과 사용자 확인 후 Import를 허용한다. not_restorable은 계속 차단한다.
Export UI Export 옵션에 일관성 모드를 추가했다. 기본값은 자동이며, strict 병렬 snapshot을 먼저 시도하고 권한이 부족하면 제한적 Export로 진행한다. 사용자는 엄격 또는 제한적을 직접 선택할 수 있다.
Python Wrapper limited Import 시 Rust Core에 strict_manifest=false를 전달할 수 있도록 RustDumpImporter.import_dump(), worker, convenience API를 보강했다.

Strict Parallel Export 동작 순서

  1. 선택된 MySQL 테이블 엔진을 검사한다.
  2. 모든 대상 테이블이 InnoDB이고 병렬 Export이면 strict parallel 경로를 시도한다.
  3. FLUSH TABLES WITH READ LOCK을 짧게 획득한다.
  4. FLUSH TABLES WITH READ LOCK이 권한 부족이면 LOCK TABLES ... READ를 시도한다.
  5. row count, range 계획을 lock 안에서 계산한다.
  6. worker connection을 미리 만들고 각 connection에서 consistent snapshot transaction을 시작한다.
  7. LOCK INSTANCE FOR BACKUP을 획득한다.
  8. 초기 global read lock을 해제한다.
  9. 미리 snapshot을 잡은 worker connection으로만 병렬 SELECT를 수행한다.
  10. 완료 후 worker transaction과 backup lock을 정리한다.

운영 전제

strict 병렬 Export는 MySQL 권한이 필요하다. 자동 모드에서는 FLUSH TABLES WITH READ LOCK, LOCK TABLES ... READ, LOCK INSTANCE FOR BACKUP 순서로 가능한 strict 경로를 시도한다. 권한이 부족하면 Export 자체를 실패시키지 않고 제한적 복원 Dump로 저장하며, manifest warning에 이유를 남긴다. 엄격 모드에서는 같은 상황을 에러로 처리한다.
